Coding with AI agents, for a thoughtful beginner.

Two tools, one mental model, and the seven habits that separate people who like the result from people who do not. Reading time: about eight minutes.

Chapter 01

Claude Code, in your Terminal.

What it is

Claude Code is Anthropic's command-line coding agent. You open a project folder in Terminal, type claude, and have a conversation with that directory. The agent can read any file, run any command, edit code in place, and commit the result. You stay the editor in chief.

Install on macOS

You will need Node.js 18 or newer. Check with node -v. If missing, install from nodejs.org or via Homebrew (brew install node). Then install Claude Code globally:

npm install -g @anthropic-ai/claude-code

A Homebrew cask also exists if you prefer that route:

brew install --cask claude-code

Verify it landed, then open it in a project:

claude --version
cd ~/projects/my-folder
claude

On first run, Claude Code opens your browser to sign in. Use a Claude Pro, Max, Team, or Enterprise account, or an API/Console account. The free Claude.ai tier does not include Claude Code.

The mental model

Your terminal becomes a conversation with the current directory. Claude can see every file in the folder, run any shell command, and edit files in place. Nothing ships unless you say yes.

Five starter prompts to try on day one

Paste these as your first messages in a fresh project. Each teaches you a different muscle.

  1. Read this folder and tell me what it does in five sentences.
  2. There is a typo somewhere on the homepage that says 'recieve'. Find it and fix it.
  3. Add a new section to README.md called 'Local setup' with these three steps: 1) clone, 2) npm install, 3) npm run dev.
  4. What changed in the last three git commits? Summarize each.
  5. I want to add a contact form. Plan the change before writing any code, then ask me before you start editing.

The commands you will actually use

Type / inside Claude Code to see the full menu.

  • /help — what the tool can do, with current keyboard shortcuts
  • /init — Claude reads the repo and writes a CLAUDE.md describing it, which loads automatically every future session
  • /clear — drop context, start fresh
  • /compact — keep going when the chat gets long; Claude summarizes and continues
  • /config — change model, theme, permission rules
  • Shift + Tab — cycle modes; one press puts you in Plan Mode, where Claude proposes a written plan and waits for your approval before touching any file
  • Esc — interrupt mid-action without ending the session

Two habits that make it work

Be specific. "Add a footer with the address, phone, copyright 2026, dark gray text, centered" gets you a usable footer. "Make the footer nicer" gets you a guess. Brief it the way you would brief a sharp new hire: outcome, constraints, examples.

Read the diff. Before Claude writes to a file, it shows you the exact change in red and green. Skim it. If something looks wrong, say so in plain English ("do not delete the old logo path, add the new one alongside it") and Claude revises. Five seconds of reading is the entire safety net.

Chapter 02

Google Antigravity, the agent IDE.

What it is

Google Antigravity is Google's agent-first development platform, launched in public preview on November 18, 2025 alongside Gemini 3. It is a desktop IDE (a modified fork of VS Code) in which autonomous AI agents plan, execute, and verify coding tasks in parallel across your editor, terminal, and a built-in browser. Different posture from a chat sidebar: you delegate, then review.

How to start on macOS

Antigravity is a downloadable desktop app, free for individuals during the public preview.

  1. Go to antigravity.google/download in Chrome.
  2. Click Download for Apple Silicon (or the Intel build if you are on an older Mac). Minimum supported macOS is Monterey (12).
  3. Open the DMG and drag the app into Applications. If macOS blocks it with "unable to verify developer," open System Settings → Privacy and Security, scroll to the blocked app, and choose Open Anyway.
  4. Launch Antigravity and sign in with your personal Google account.
  5. Approve the prompts for terminal, file system, and browser automation access. Chrome should be installed; agents drive it for verification.

There is no separate signup form. Your Google account is the account.

The mental model

Chatting with a model is a conversation. You ask, it answers, you copy code, you run it. Antigravity inverts the posture. You assign a task, then walk away while one or more agents do the work in their own workspace.

Two views matter. The Editor view looks like VS Code with an agent sidebar, useful when you want to stay close to the code. The Manager view is mission control: you spawn agents, watch them run in parallel, and switch between them like browser tabs.

Agents do not just emit code. They produce Artifacts: an implementation plan, a task list, a diff, screenshots, a recorded browser walkthrough proving the feature works. You review Artifacts the way you review a Google Doc, leaving inline comments. Approvals happen at gates you set (Always Proceed, Agent Decides, Request Review), so you stay in the loop for anything that touches money, secrets, or production.

When to reach for it vs Claude Code

Reach for Antigravity when:

  • The task is visual or web-based and you want the agent to load the page in a browser and prove it works.
  • You want to spawn two or three agents on the same problem (different stacks, different approaches) and compare their Artifacts side by side.
  • You prefer a graphical workspace with diffs, plans, and recordings laid out for you.
  • The job is large enough that you would rather kick it off, get coffee, and come back to review.

Reach for Claude Code when:

  • You are in a terminal already and want a single, coherent conversation about your repo.
  • The edit is surgical: rename a function across files, write a quick script, fix one failing test.
  • You want speed and low ceremony. No separate IDE, no approval gates.
  • You are operating over SSH on a server with no GUI.

Honest summary: Antigravity is better for parallel exploration and verifiable UI work. Claude Code is faster and more focused for everyday edits and ops work.

One concrete starter workflow

Open Antigravity, click Manager, then Start Conversation. Pick Gemini 3 Pro and Planning mode. Paste:

Build me a one-page personal site for Frank Heijckers.
Three sections: a short bio, three career highlights, and
a contact form that emails me. Use plain HTML and Tailwind.
When it works, open it in the browser and show me a recording.

What happens next: the agent produces a plan Artifact (file structure, sections, dependencies). You comment on it the way you would on a Google Doc ("make the bio two paragraphs, not three"). It writes the files, runs them, opens the page in its built-in browser, and attaches a screen recording as an Artifact. You either approve, or redirect with another comment. The agent keeps going until the Artifact matches the brief.

One gotcha

Rate limits are the loudest complaint in current coverage. The free tier can lock you out for up to seven days after 20 to 30 minutes of heavy use, and the meter is opaque (it shows "compute effort," not tokens, and can read 100 percent right before a multi-day cooldown). Even the paid tier reports 6 to 10 day cooldowns on specific models. Treat Antigravity as an exploration and prototyping tool for now, and keep Claude Code in your back pocket for when the meter runs out.

Chapter 03

How to work well with either.

Seven principles. Each is a directive followed by a short paragraph. They apply whether you are in a terminal or in Antigravity's Manager view.

1

Plan before you code.

Describe the outcome before asking for code. "I want a contact form that emails me when filled, validates an email address, and shows a thank-you message" beats "build me a contact form." The agent fills in everything you do not specify, and you may not love its choices. A minute spent describing what you want is the cheapest minute in the whole project.

2

Specify the constraints, not the steps.

Tell it what must be true at the end, not how to get there. "The new endpoint must return JSON, must not change the existing /users response shape, and must be tested" gives the agent room to do its best work. Step-by-step instructions waste its intelligence and often box it into a worse solution than it would have found on its own. Brief it like a sharp junior: set the bar, name the non-negotiables, then get out of the way.

3

Read what it writes.

Claude or Antigravity will show you a diff or a list of changes. Skim it before you say yes. You do not need to understand every line; you need to confirm that the change matches your intent. Two minutes of reading saves an hour of debugging, and you will start to absorb the shape of the code by osmosis.

4

Iterate in small steps.

One change at a time beats one big request. "Add the contact form," then "now add validation," then "now style it" beats "add a styled, validated contact form with seven features." Smaller asks fail less often, are easier to review, and are trivial to roll back. Big requests hide small mistakes; small requests surface them.

5

Use git, always.

Commit before any AI change. If the result is wrong, git reset --hard returns the file to safety in one second. Without git, a confused agent can rewrite a file you cared about and you have no undo button. Claude Code and Antigravity both respect git, but only if you have set it up. Treat the commit as the seatbelt you click before each drive.

6

Trust, but verify.

If the agent says "I added the feature and tested it," run the feature yourself. Open the page. Click the button. Send the form. AI is confident; reality is the judge, and the confidence does not correlate with correctness. A working demo in your own browser is worth more than ten reassuring paragraphs in the chat.

7

Curiosity is the new skill.

When the agent does something unfamiliar, ask why instead of skipping. "Why did you choose that library?" "What does this function do?" "Is there a simpler way?" The conversation is the learning. In two months of curious use, you will have a working sense of modern code without ever sitting through a course, because every real project becomes its own seminar.

The skill here is not coding. It is being a clear-eyed editor of someone else's work: setting the brief, reading the draft, catching the gap between what was promised and what was delivered. You have been doing exactly that kind of work for decades. That is the leverage.
